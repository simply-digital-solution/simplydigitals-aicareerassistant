AI Career Assistant — System Workflow

From job discovery to pipeline — end-to-end

Click any step to expand details

🌐
1 · Scrape
⏰ Daily Cron · 07:00 SGT
APScheduler fires scrape_for_all_users() every morning at 07:00 Asia/Singapore. Also triggerable manually via the Refresh button.
🔍 MCF API Search
Queries MyCareersFuture API once per target job title in the user's profile. Retrieves up to 100 results per title including the original posting date.
🏭 Industry Filter
Each result is scored against the user's target industries using keyword similarity. Jobs below 80% match are dropped before storage.
🔄 Deduplication
Skips any job already stored for this user with the same title + company + posting date. Prevents duplicate cards from multiple scrape runs.
💾 Store Job
New jobs inserted into job_postings with scored=0. The background scorer loop picks them up automatically.
→
🤖
2 · Score
♾️ Continuous Loop
Background asyncio task runs indefinitely. Picks the oldest scored=0 job, scores it, then sleeps 30 seconds before the next one. Sleeps 5 minutes when the queue is empty.
👤 Load Profile
Reads the user's full profile — all skills (no cap), years of experience, target titles, industries, seniority, and location preferences.
💬 Inject Feedback
Top 10 thumbs-up / thumbs-down entries from job_feedback are appended to the prompt as few-shot calibration examples before the LLM call.
🧠 DeepSeek-R1:7b
Full job description (not truncated) + profile sent to DeepSeek-R1:7b via Ollama. Returns a JSON object with fit score, reasons, risks, keywords, and a 5-category breakdown.
📊 Score Breakdown
Five fixed categories scored 1–10: Technical Skills, Experience Level, Domain/Industry, Seniority, Location & Work Mode. Stored as JSON in scoring_breakdown.
✅ Write Back
Updates job_postings with scored=1, fit_score, reasons, risks, key_keywords, and scoring_breakdown.
→
📋
3 · Review
🎯 Fit Score Badge
Each job card shows a colour-coded badge: green ≥70%, amber 50–69%, red below 50%. Unscored jobs show "Scoring…" until the loop processes them.
📈 Score Breakdown Table
Expandable table showing all 5 categories with what the JD requires vs what the user brings, plus a 1–10 score per category. Green ≥8, amber 5–7, red ≤4.
🔎 Filters
Filter by role (editable combobox with profile titles), posting date (7 / 14 / 30 / 90 days), and fit score threshold (All / 50%+ / 60%+ / 70%+ / 80%+). Unscored jobs always visible.
🔁 Re-score
Circular-arrow button resets scored=0 on the job. It re-enters the scorer queue immediately and is re-evaluated with the latest profile and accumulated feedback.
🗄️ Archive
Marks the job as archived=1. It disappears from the Research Panel permanently. Useful for roles that are clearly not relevant.
→
💬
4 · Feedback
👍 Mark Relevant
One-click thumbs-up saves a relevant entry to job_feedback. The card border turns green. This job will appear as a positive example in future LLM prompts.
👎 Mark Not Relevant
Thumbs-down opens an inline reason picker: Wrong industry, Wrong seniority, Too junior/senior, Wrong location, Already applied, Company not a fit. Reason is stored and shown on the card.
🎓 Few-shot Injection
Up to 5 relevant + 5 not-relevant feedback rows are prepended to every future scoring prompt. The LLM uses these as calibration — adapting scores to match the user's demonstrated taste over time.
→
🗂️
5 · Pipeline
📌 Save to Selected
Bookmark button on a job card creates an Application record at Selected status. The job now appears on the Kanban pipeline board.
📝 Applied
Drag the card to Applied once you've submitted your application. Track the date and source URL.
🎙️ Interviewing
Move to Interviewing when you receive a callback. Interview prep agent available for role-specific question coaching.
🏆 Offered
Offer received. Track the outcome — Offered, Rejected, or Withdrawn — to maintain a full history of your job search.
Key Feedback Loops
🔄

Scrape → Score (automatic)

New jobs land as scored=0. The background scorer picks them up within 30 seconds — no user action needed.

💬

Feedback → Score (self-calibrating)

Every thumbs-up / thumbs-down is injected into the next LLM prompt. The scorer learns from the user's taste and improves over time.

🔁

Re-score → Score (on demand)

After updating your profile skills or giving new feedback, hit Re-score to force any job to be re-evaluated with the latest context.
